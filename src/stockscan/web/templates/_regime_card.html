{% endmacro %} {% set vol_value %} {% if regime.vix_level is not none %}{{ "%.2f"|format(regime.vix_level|float) }}{% else %}—{% endif %} {% endset %} {% set vol_explanation %}

Volatility regime. Inverted percentile rank of the VIX index over the trailing 252 trading days. The headline number on the left is today's spot VIX; the rank shows where that level sits in its own recent history. Low VIX (calm) → low rank → high score; spiking VIX (panic) → high rank → low score. We use a percentile rank instead of fixed thresholds because what counts as "stressed" drifts over time — VIX in the high teens was elevated in 2017 and normal in 2022.

  • Window: trailing 252 trading days (~1 year).
  • Score formula: 1 - rolling_pct_rank(VIX, 252).
  • Source: EODHD /eod/VIX.INDX, stored in the bars hypertable.

Trend regime. Smooth combination of two SPY trend signals: today's close relative to its 200-day SMA (the headline value on the left) and the 20-day slope of the SMA itself. Each is clipped to a saturation band (±5% for position, ±2%/20d for slope) and averaged into a [0, 1] score where 0.5 is neutral. Captures both where the market is and which way it's moving.

  • SMA(200) 20-day slope: {% if regime.spy_sma200_slope_20d is not none %} {{ "%+.2f"|format(regime.spy_sma200_slope_20d|float * 100) }}% over the last 20 days {% else %} not persisted (pre-0011 row) {% endif %}
  • SPY close: {{ "%.2f"|format(regime.spy_close|float) }} · SMA(200): {{ "%.2f"|format(regime.spy_sma200|float) }}
  • No percentile rank — this is a position+slope blend, not a percentile-based score.

Breadth regime. Cheap proxy for "are most stocks participating?" using the ratio of RSP (equal-weight S&P 500) to SPY (cap-weight S&P 500). When the 20-day SMA of that ratio is above its 200-day SMA, breadth is broadening — small/mid caps are keeping up with mega-caps. When it's below, breadth is narrowing (the "Mag 7 carrying everything" regime). The headline value on the left is the relative gap between those two SMAs.

  • Spot RSP/SPY ratio: {% if regime.rsp_spy_ratio is not none %} {{ "%.4f"|format(regime.rsp_spy_ratio|float) }} {% else %} not persisted (pre-0011 row) {% endif %}
  • Saturation band: ±5% gap → score in [0, 1] linearly.
  • No percentile rank — this is a ratio-trend signal, not a percentile-based score.
{% endset %} {{ component_row("Breadth", "RSP/SPY 20d–200d gap", breadth_value, None, regime.breadth_score, "20%", breadth_explanation) }} {% set credit_value %} {% if regime.hy_oas_level is not none %}{{ "%.2f"|format(regime.hy_oas_level|float) }}%{% else %}—{% endif %} {% endset %} {% set credit_explanation %}

Credit regime. Inverted percentile rank of HY OAS (ICE BofA US High Yield Option-Adjusted Spread, FRED series BAMLH0A0HYM2) over the trailing 252 days. Tight spreads = high score; wide spreads = low score. Bond markets typically lead equities by 1–3 days when stress is brewing, which is why this is in the composite at all despite a daily-bar equity system.

  • HY OAS z-score: {% if regime.hy_oas_zscore is not none %} {{ "%+.2f"|format(regime.hy_oas_zscore|float) }} (rolling 252-day mean / std) {% else %} — {% endif %}
  • Credit-stress flag (separate, discrete): fires when rank > 0.85 and spreads are rising over the last 5 days. When set, all sizes are halved and new long entries are blocked entirely — that's the tail-risk circuit breaker, orthogonal to this smooth credit_score.
  • Score formula: 1 - rolling_pct_rank(HY OAS, 252).

Legacy ADX/SMA — kept at 0% weight. The v1 regime classifier used only ADX(14) and SPY's position relative to its 200-day SMA to bucket the market into one of four discrete labels. The v2 composite (rolled out in migration 0010) replaces that with the four percentile-rank components above. We keep the v1 fields for two reasons:

  • The discrete regime label on the header strip (trending_up / trending_down / choppy / transitioning) is still derived from this ADX + SMA(200) classifier — it's the only signal in the system that produces a categorical bucket rather than a continuous score, and the strategy regime_affinity mappings key on those labels.
  • Back-compat: pre-0010 rows in market_regime only have these two fields populated. Removing them from the UI would erase historical context for anything dated before the v2 rollout.

The weight column reads 0% deliberately: this row contributes nothing to the composite score above. Treat it as descriptive context, not as a signal that competes with the active components.
